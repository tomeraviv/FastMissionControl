Open source macOS Mission Control replacement

Mission Control is too slow.

Fast Mission Control is a lightweight overview built to fix Apple's laggy and slow Mission Control, especially on 4K+ monitors, 5K displays, ultrawide screens, and multi-monitor setups where the stock animation often feels heavy.

If you found this from Google or Reddit 
defaults write com.apple.dock expose-animation-duration -float 0.1

People still share that command when answering questions like "How to speed up animations on macOS" and "Speed up Mission Control animations in macOS". On current macOS versions, that old Dock tweak no longer fixes the real Mission Control slowdown that many users still hit every day.

What this project does instead

Fast Mission Control opens its own fast overview, keeps windows grouped by display, renders previews, and makes window selection feel immediate instead of waiting on Apple's built-in Mission Control animation path.

  • Built for high-resolution and multi-display Macs
  • Open source and easy to inspect in Xcode
  • Focused on practical speed, not old terminal myths

What Fast Mission Control can do right now

Fast overview

Opens a custom Mission Control-style overview with tuned animation timings instead of relying on the laggy default experience.

Multi-monitor aware

Preserves per-display layout so stacked or clustered windows still spread out across the right monitor.

Window previews

Uses cached still previews and live previews so the overview stays readable when a lot of windows are open.

FAQ for searchers

How to speed up animations on macOS?

If your real pain point is Mission Control, the old Dock defaults commands are outdated advice. Fast Mission Control exists specifically to speed up the Mission Control experience itself.

Speed up Mission Control animations in macOS

That search usually lands on stale blog posts and Reddit comments. This project is the modern, open-source alternative for users who still see Mission Control lag on current macOS builds.

Who is this for?

Mac users with large displays, 4K+ monitors, multiple monitors, heavy multitasking workflows, and too many windows open at once.

What permissions does it need?

Screen Recording and Accessibility. Those are required so the app can read window state, show previews, and switch to the right window instantly.
